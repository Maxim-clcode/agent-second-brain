## ШАГ 3 — Определи загруз на ближайшие недели

1. Подсчитай суммарное время всех задач проекта (в часах).
2. Раздели на 6 свободных часов в день (рабочий день 9:00–18:00 минус 1ч обед = 8ч, но обычно ~6ч свободных).
3. Так ты получишь примерное число рабочих дней N.
4. Для каждого рабочего дня от завтра до горизонта N+5 дней — вызови `get_tasks_by_date(YYYY-MM-DD)`.
   - Пропускай выходные (сб, вс) и праздники РФ.
   - Обед 13:00–14:00 всегда заблокирован.

Построй карту занятости: день → список занятых интервалов [HH:MM–HH:MM].

**Важно:** задачи из TickTick с тегом `platrum` — это жёсткие назначения из внешней системы.
Их нельзя двигать и нельзя ставить поверх них задачи проекта. Они всегда блокируют слот.

## ШАГ 5 — Расставь задачи проекта в слоты

Сортировка: P1 → P2 → P3 → P4, внутри приоритета — по дедлайну (ближайший вперёд).

Правила расстановки:
- Задачи ставь вплотную друг к другу — без пауз между ними
- Нельзя занимать 13:00–14:00 (обед)
- Нельзя выходить за пределы 09:00–18:00
- Если задача не влезает в остаток дня — переноси на следующий свободный день
- Нельзя ставить задачи в выходные и праздники РФ

Запомни для каждой задачи: дата, startTime, endTime.

## ШАГ 8 — После подтверждения

**8а. Добавь все задачи в TickTick**
Для каждой задачи вызови `add_task` с параметрами:
- `title` = название задачи
- `startDatetime` = YYYY-MM-DDTHH:mm (локальное время Владивостока)
- `endDatetime` = YYYY-MM-DDTHH:mm

**8б. Обнови статус в Notion — СТРОГО по датам**

Сначала вычисли даты текущей недели:
- `WEEK_START` = ближайший понедельник ≤ сегодня (в формате YYYY-MM-DD)
- `WEEK_END` = `WEEK_START` + 4 дня (пятница, YYYY-MM-DD)

Для КАЖДОЙ задачи из дорожной карты:
- Если `date` задачи >= `WEEK_START` И <= `WEEK_END` → обнови статус на **«Спринт недели»** через `notion-update-page`
- Если `date` задачи > `WEEK_END` (следующие недели) → статус **оставь «Бэклог»**, не трогай

**Запрещено переводить задачи будущих недель в «Спринт недели».**
Бот переведёт их сам при наступлении нужной недели (через CLAUDE.md логику планирования).

## При запросе перестройки дат («не успею», «ухожу в отпуск», «сдвинь задачи»)

**Триггеры:** «не успею», «не смогу выполнить», «сдвинь», «отпуск», «болею», «форс-мажор», «перенеси», «перестрой даты», «сдвинь всё»

**Действие:**

1. Уточни:
   - По какому проекту? (если несколько roadmap-файлов)
   - С какой даты сдвинуть? (по умолчанию — сегодня)
   - На сколько дней? (или: «до какого числа у тебя [отпуск/форс-мажор]?»)

2. Посчитай количество рабочих дней сдвига и покажи Максиму:
   ```
   📅 Сдвигаю дорожную карту: [название проекта]
   • Старт сдвига: ДД.ММ
   • Сдвиг: +N рабочих дней
   • Новый финиш: ~ДД.ММ.ГГГГ
   
   Подтверждаешь?
   ```

3. После подтверждения — запусти скрипт:
   ```bash
   /home/brain/projects/agent-second-brain/.venv/bin/python \
     /home/brain/projects/agent-second-brain/scripts/shift_roadmap.py \
     --project [slug] \
     --shift N \
     --from-date YYYY-MM-DD
   ```
   Slug = имя файла без `.json` из папки `data/roadmaps/`.

4. После запуска скрипт сам:
   - Обновит даты в TickTick
   - Обновит статусы в Notion
   - Сохранит новый JSON
   - Обновит закреп в Telegram
   - Отправит уведомление

5. Скажи Максиму: «✅ Дорожная карта перестроена. Все задачи сдвинуты на N дней, новый финиш — ДД.ММ»
